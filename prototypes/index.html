Abstract

Procbar groups and kills local dev processes by git worktree. The visual direction is "instrument panel": quiet monochrome, a single amber accent that emerges as processes work, hairline structure, tabular mono numerics.

This document renders every surface the v1 app shows: the menu bar icon, the popover in dark and light, its empty and error states, the stop-button state machine, and the settings window. The signature element to watch is the 2px amber CPU bar under each process name — the thing you'll remember.

A custom 18×18pt template image — three horizontal bars of varying length, reads as an equalizer or a level meter. Drawn with system tint so it inverts correctly in both menu bar themes. No colored status dot on the icon itself; activity is conveyed by opacity.

FIG 06

Stop button · state machine

Single button, four resolved states. The amber arc during terminating is the 3-second grace window; a second click during that window fast-paths to SIGKILL.

IDLE
Resting. Hover fills warning-red. Click → SIGTERM to process tree.
TERMINATING
SIGTERM sent. Arc winds for 3s. Click again = immediate SIGKILL.
KILLING
Grace expired (or fast-path hit). SIGKILL fires, red flash 120ms.
DONE
Process exited. Green tint 200ms before row collapses.

FIG 10

Activity states · dot legend

Each process row carries a 5pt dot left of the name indicating whether the process is doing work right now, worked recently, or has been quiet. Resolved from a rolling lastActiveAt timestamp updated every poll. Idle rows also swap their uptime readout for idle Xm to highlight stale dev servers.

IDLE · No activity for longer than the recent window. Dot is tertiary gray; the action column shows idle 3h instead of uptime.
TRANSITIONS
Any sample with CPU > threshold writes now to lastActiveAt[pid]. State is a pure function of (now − lastActiveAt) vs the recent window.
FIRST SIGHT
A newly-tracked PID is seeded with lastActiveAt = now so a fresh dev server never displays as idle the first time it appears.
